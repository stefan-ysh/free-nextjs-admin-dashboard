# Development Roadmap (Current)

更新日期：2026-02-18

该路线图基于当前业务范围：采购审批 + 到货入库 + 独立报销 + 财务打款 + 预算调整。

## P0（已完成）
- 采购与报销流程解耦。
- 采购不再走财务打款节点。
- 报销支持关联采购，且过滤掉对公转账采购。
- 财务付款处理页改为“报销待打款”口径。
- 待办/已办统一逻辑（按当前角色是否需要操作来划分）。
- 预算调整单独入口，自动落账到收支流水。
- 废弃模块删除：项目、客户、日程、采购付款分表。

## P1（进行中）
- 文档去旧（IA、数据模型、测试清单）与代码口径统一。
- 角色权限口径继续收敛（超级管理员只读兜底）。
- 报销附件与预览体验优化（文件名展示、内嵌预览稳定性）。

## P2（下一步建议）
- 增加关键流程自动化回归脚本：
  - 员工采购 -> 审批 -> 入库
  - 直接报销 -> 财务打款
  - 采购关联报销 -> 财务驳回 -> 申请人补充 -> 再提交
- 为“采购单唯一关联报销”增加数据库唯一约束（或事务级防重）。
- 增加健康检查命令到 CI（如 `scripts/check-purchase-health.ts`）。

## 非目标（当前不做）
- 项目管理、客户管理、日程管理相关功能。
- 采购付款分表与采购财务打款流程。

## 里程碑验收口径
- 员工看见：自己的采购/报销 + 自己待办。
- 审批管理员看见：采购审批待办 + 采购审批已办。
- 财务看见：报销待打款 + 打款已办。
- 超级管理员：只读查看全量数据，不参与流程动作。
