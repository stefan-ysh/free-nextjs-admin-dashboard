# 业务规范基线（产品/运营/测试）

## 1. 角色定义
- 超级管理员：仅查看全部数据，不参与审批/打款。
- 审批管理员：仅负责采购审批。
- 学校财务/单位财务：仅负责报销处理（打款或驳回）。
- 员工：发起采购、入库、发起报销、处理驳回补料。

## 2. 采购流程
- 标准流程：`采购申请 -> 审批管理员审批 -> 申请人入库`
- 审批通过后状态：`待入库 (pending_inbound)`
- 采购流程无独立财务打款节点。

## 3. 报销流程
- 可直接报销，也可关联采购单报销。
- 关联采购单需满足：
  - 非对公转账
  - 已入库
  - 未被其他报销单占用
- 财务处理时可：
  - 打款完成
  - 驳回（申请人补料后重提）

## 4. 待办/已办口径
- 待办：当前角色“需要执行动作”的事项。
- 已办：当前角色“已执行过动作”的事项。
- 草稿不计入待办。
- 全员统一只保留“待办/已办”，不再使用“在途”。

## 5. 页面与文案要求
- 所有错误提示必须中文且可行动。
- 操作人字段显示姓名，不显示 UUID。
- 详情页默认只读，编辑场景才允许修改。
- 财务动作按钮建议统一命名“去处理”。

## 6. 发布验收重点
1. 学校单只流向学校财务，单位单只流向单位财务。
2. 被驳回报销会进入申请人待办。
3. 待办角标数量与列表一致。
4. 菜单只允许单一路由高亮。
5. 报销状态全部中文展示。
